## GITFLOW

É uma forma de organização do trabalho para ambientes de programação, utilizando o versionamento de código.

O GitFlow funciona, de forma resumida, da seguinte forma:

1. Criamos uma branch chamada 'develop', da qual funcionará como uma main 'provisória', assim não agindo diretamente
na main de nosso diretório.
2. A partir disso, tratamos a develop como main quando fizermos os commits.
3. Seguindo a lógica, geramos outras branch's para evitar bagunçar nossa main provisória, identificando-as com um
determinado padrão.
4. Quando finalizarmos o projeto, aí sim commitamos a develop para a main (sem apagar a develop, pois sempre pode
haver o que melhorar do projeto)

Um exemplo de padronização de branch's:

Nomes de branches são compostos de 3 partes:

1 — type ou categoria do branch. Os types podem ser os seguintes:

docs: apenas mudanças de documentação;
feat: uma nova funcionalidade;
fix: a correção de um bug;
perf: mudança de código focada em melhorar performance;
refactor: mudança de código que não adiciona uma funcionalidade e também não corrigi um bug;
style: mudanças no código que não afetam seu significado (espaço em branco, formatação, ponto e vírgula, etc);
test: adicionar ou corrigir testes.

2 — o que o branch faz em si ou o arquivo ao qual vamos mexer.

3 — Código da tarefa no Jira. Ex.: PL-123.

## Importância do GitIgnore para o versionamento

Dentro do arquivo '.gitignore', adicionamos majoritariamente os dados que estamos trabalhando. O git é utilizado
majoritariamente para versionamento de arquivos de texto, como códigos de programação e arquivos '.md' ou '.txt'.

Por isso, dentro do arquivo do GitIgnore é comum destacarmos de forma geral:

*.pptx
*.csv
*.xls
*.xlsx
*.parquet

Pois esses são os formatos de arquivos que comumente estão guardados nossos dados.
